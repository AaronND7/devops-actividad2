💭 PARTE 10: REFLEXIÓN
======================

✅ ¿Qué ventaja tienen los commits progresivos?
Los commits progresivos facilitan el tracking de cambios, permiten rollback a puntos específicos, mejoran la colaboración entre desarrolladores y hacen el código más mantenible y comprensible. Cada commit representa una funcionalidad específica que puede ser testeada y validada independientemente.

✅ ¿Por qué evitar hardcoding?
Evitar hardcoding permite reutilización del código en diferentes entornos, facilita cambios sin modificar el código fuente, mejora la seguridad al separar configuración sensible y permite diferentes configuraciones para desarrollo/producción sin cambios en la lógica.

✅ ¿Qué rol cumple deploy.sh?
deploy.sh actúa como orquestador del flujo CI/CD simulado, integra múltiples scripts (EC2 + S3), maneja errores y genera logs centralizados, e implementa el flujo DevOps completo desde la ejecución hasta el despliegue.

✅ ¿Qué ventaja tiene separar config del código?
Separar config del código permite cambios de configuración sin recompilar, facilita la gestión de diferentes entornos, mejora la seguridad al separar datos sensibles y permite versionado independiente de configuración vs código.
